# StudLMS factcheck — план по красным пунктам

Рабочее правило:
- Делаем по одному пункту.
- Сначала находим файл/класс/место.
- Даём маленькую правку.
- Проверяем на хосте.
- Только после подтверждения фиксируем в Git.
- Общие элементы сайта правим в общих стилях (`theme/css/style.css`), обучение — в `theme/css/training.css`.
- Не добавлять новые временные CSS-файлы без необходимости.

## Статусы

- `[done-host]` — исправлено на хосте, нужно зафиксировать в Git после подтверждения.
- `[todo]` — не готово.
- `[hold]` — спорно/не критично/нужно решение.

### PC-01 — левое меню: раскрытый раздел `Обучение`

Статус: `[done-host]`

Что было в документе:
- при открытом боковом меню стрелка смотрит вниз, должна вверх;
- тайтл `Обучение` и иконка должны быть по дизайну;
- иконка отличается по форме;
- тайтл `Обучение` должен быть Regular, как пункты меню;
- иконка шапки должна быть двухцветная.

Что уже сделано:
- направление стрелки исправлено на хосте в общем стиле.

Где фиксировать:
- `theme/css/style.css`
- блок `.nav-item.menu-toggle .menu-button .button-menu`

Проверенная логика:
```css
.nav-item.menu-toggle .menu-button .button-menu {
    transform: rotate(180deg);
}

.nav-item.menu-toggle.active .menu-button .button-menu {
    transform: rotate(0deg);
}
```

Осталось по этому пункту:
- проверить/довести цвет/градиент/начертание активного пункта `Обучение`;
- проверить форму иконки `Обучение`;
- отдельно проверить двухцветную иконку шапки.
